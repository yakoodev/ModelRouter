# Задача 04 — Auth strategies v1

## Цель
Реализовать базовые стратегии авторизации для production/dev сценариев.

## Инструкция
- Реализовать `NoAuth`, `ApiKeyAuth`, `BearerAuth`, `CustomHeadersAuth`.
- Применение к `HttpRequestMessage` сделать единообразным.
- Добавить unit-тесты для всех стратегий.
- После завершения задачи и мержа PR перенести этот файл из `docs/tasks/todo/` в `docs/tasks/done/`.

## Критерии приемки
- Все стратегии корректно выставляют заголовки.
- API стратегий расширяемо без изменений клиентского кода.
- `dotnet test MultiLlm.slnx` проходит.
